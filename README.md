Resumen Diseño lógico de bases de datos.  
El diseño lógico de una base de datos consiste en pasar lo que ya se había planteado en el diagrama entidad-relación a tablas del modelo relacional. Básicamente, cada entidad se convierte en una tabla y sus atributos pasan a ser columnas, incluyendo su clave primaria.
Las relaciones entre las entidades también se transforman según el tipo de relación. Por ejemplo, cuando es de muchos a muchos se crea una tabla adicional, y cuando es de uno a muchos se agrega una clave foránea en la tabla correspondiente.
La idea es organizar bien la información para que los datos estén conectados correctamente y no haya desorden. Así, el modelo final en tablas representa de forma clara lo que el sistema necesita guardar.
